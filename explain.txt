Claro! Vou explicar detalhadamente do que se trata esse projeto.
Este é um projeto relacionado ao setor de saúde suplementar (planos de saúde) que envolve dados da ANS (Agência Nacional de Saúde Suplementar), que é o órgão regulador dos planos de saúde no Brasil.
O projeto se divide em 3 partes principais:
Parte 1 - Web Scraping
O objetivo é coletar documentos importantes da ANS
Especificamente, você precisa baixar dois documentos principais:
Anexo I: que contém o Rol de Procedimentos e Eventos em Saúde (lista de todos os procedimentos que os planos de saúde são obrigados a cobrir)
Anexo II: que contém as Diretrizes de Utilização (regras para uso dos procedimentos)
Depois, estes documentos precisam ser compactados em um único arquivo
Parte 2 - Transformação de Dados
Aqui o foco é trabalhar com o conteúdo do Anexo I
Você precisa:
Extrair a tabela do Rol de Procedimentos do PDF
Converter essa informação para um formato mais útil (CSV)
Fazer algumas correções nos dados, como expandir as abreviações:
OD (que provavelmente significa "Odontológico")
AMB (que provavelmente significa "Ambulatorial")
Parte 3 - Banco de Dados
Esta parte trabalha com dados financeiros e cadastrais das operadoras de planos de saúde
Envolve dois conjuntos de dados:
Demonstrações contábeis: dados financeiros dos últimos 2 anos das operadoras
Cadastro de operadoras ativas: informações sobre as operadoras que estão funcionando
Você precisa:
Baixar esses dados
Criar um banco de dados estruturado
Importar todos os dados mantendo a integridade das informações
A importância deste projeto está em:
Organizar informações cruciais sobre cobertura de planos de saúde
Permitir análise de dados financeiros das operadoras
Criar uma base de dados estruturada para consultas e análises
Pelo que vejo no seu config.py, você já tem uma estrutura inicial configurada com:
URLs da ANS
Estrutura de diretórios para organizar os dados
Configurações para trabalhar com arquivos CSV
Configuração de um banco de dados PostgreSQL
Uma estrutura de API e frontend (sugerindo que você planeja criar uma interface para visualizar esses dados)
Por onde você gostaria de começar? Podemos:
Começar pelo web scraping para coletar os documentos
Trabalhar na extração dos dados dos PDFs
Ou estruturar o banco de dados primeiro
